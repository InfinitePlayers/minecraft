Minecraft Server Safety Tutorial

Step 1 of 1 Goal: backups 08:00–22:00, delete after 48h, whitelist, no ops, rules shown

Step 0, File Explorer basics

What is File Explorer
File Explorer is the app you use to find files and folders on your computer.

How to open File Explorer

Try any of these:
  1. Press Windows key + E
  2. Click the folder icon on the taskbar
  3. Press the Windows key, type File Explorer, press Enter

Show file extensions

File extensions are the bit after the dot, like .bat or .txt.

Turn them on:
  1. Open File Explorer
  2. Click View
  3. Click Show
  4. Tick File name extensions
This matters because Windows can hide important info.
You want to see backup_world.bat not just backup_world.

Common file types

  • .txt writing
  • .html web page
  • .bat Windows commands you can run
  • .exe program
  • .json data lists, Minecraft uses it
  • .properties settings, Minecraft uses it
  • .jar Java program file, Minecraft server uses it
Why we start here
If you can open File Explorer and spot file extensions, you can do everything else safely.

You need an app to edit text files. You can use:
  • Notepad, already installed, good enough
  • Notepad++, easier for code and learning

How to open Notepad

Step 2, Find your Minecraft server folder

Your server is on the Desktop right now. That is ok. We must find the exact folder path.

How to find it

  1. Open File Explorer
  2. Click Desktop on the left
  3. Find your server folder, click it once

How to copy the folder path

  1. At the top of File Explorer, click the address bar
  2. It turns into text
  3. Press Ctrl + C to copy
  4. Paste it into a note using Ctrl + V
Example: 
C:\Users\YOURNAME\Desktop\MinecraftServer

Find the world folder name

Double click into the server folder.
Look for a folder called world.
If you see it, the world folder name is world.

Find the server jar file name

In the server folder, look for a file ending in .jar.
Examples:
  • server.jar
  • paper-1.xx.x.jar
  • spigot-1.xx.x.jar
Write down its exact name.
Check
You now know:
  • Server folder path
  • World folder name
  • Jar file name

Step 3, Create the backup and tools folders

In File Explorer:
  1. Click This PC on the left
  2. Double click Local Disk (C:)
  3. Right click in empty space, New, Folder
  4. Name it MinecraftBackups
  5. Make another folder called MinecraftTools
Why we do this
Backups should not live inside the server folder. That reduces mistakes.

Step 4, Show rules before players join (server list message)

This message shows in the server list before someone clicks Join. It is called the MOTD.
Open this file in Notepad or Notepad++:
(your server folder)\server.properties
Example:
C:\Users\YOURNAME\Desktop\MinecraftServer\server.properties
Find the line that starts with motd= and replace it with this: 
motd=Dexter’s Server. Respect builds. No stealing. Open 08:00 to 22:00. Backups active.
Why we do this
It sets expectations before they join, so there is less arguing later.
Save the file. Restart the server later so it updates.

Step 5, Show rules when players join (in chat)

This message appears in game chat when a player joins. There are two ways:
  • Easy and best: use a plugin by running a Paper server
  • Harder: vanilla options are limited

Quick check, do you already have Paper

Look at your server jar file name in the server folder:
  • If it starts with paper- you already have Paper
  • If it is server.jar you are probably running vanilla

Option A, Paper server plus a plugin (recommended)

What this gives you
Every time someone joins, they see the rules.
How plugins work
Paper can load extra features called plugins. Plugins live in a folder called plugins inside your server folder.
Steps to add a join rules message using EssentialsX:
  1. Stop the server
  2. Open your server folder
  3. If there is no folder called plugins, create it
  4. Download the EssentialsX plugin file, it will be a .jar
  5. Put that .jar into the plugins folder
  6. Start the server once, then stop it again
  7. A folder called Essentials will appear inside plugins
  8. Open plugins\Essentials\config.yml in Notepad++
  9. Find the motd: section and set it like this
Example EssentialsX join rules message (this is inside config.yml): 
motd:
  - "&aWelcome to Dexter’s Server"
  - "&fRules"
  - "&71. Do not break other people’s builds"
  - "&72. Do not take items without asking"
  - "&73. Keep chat respectful"
  - "&eServer hours: 08:00 to 22:00"
  - "&cGriefing means rollback and ban"
Check
Start the server. Join the server. You should see the rules in chat.
Important
Plugins require Paper or Spigot. Vanilla servers do not run plugins.

Option B, vanilla server (limited)

Vanilla can easily show the server list message (Step 4). Vanilla does not have a simple built in join rules message. You can do it with advanced command setups, but it is more complex than the rest of this tutorial.
Why plugins are worth it
If your main problem is friends joining and pushing rules, Paper plus a join rules plugin makes it simple and reliable.

Step 6, Create the hourly backup script

Open Notepad or Notepad++.

Create a new file and save it as:
C:\MinecraftTools\backup_world.bat
Paste the code below. Then edit ONLY the 2 lines at the top. 
@echo off
setlocal

REM CHANGE THESE TWO LINES
set SERVER_DIR=C:\Users\YOURNAME\Desktop\MinecraftServer
set WORLD_DIR=world

set SRC=%SERVER_DIR%\%WORLD_DIR%
set DEST=C:\MinecraftBackups

for /f %%i in ('powershell -NoProfile -Command "Get-Date -Format yyyy-MM-dd_HH-mm"') do set TS=%%i

if not exist "%DEST%" mkdir "%DEST%"

xcopy "%SRC%" "%DEST%\%WORLD_DIR%_%TS%" /E /I /H /C /Y >nul

echo Backup created: %DEST%\%WORLD_DIR%_%TS%
endlocal

How to save it correctly in Notepad

  1. File, Save As
  2. Save as type: All Files
  3. File name: backup_world.bat
  4. Save it into: C:\MinecraftTools
Test it now
Open File Explorer, go to C:\MinecraftTools and double click backup_world.bat.
Then check C:\MinecraftBackups and you should see a new backup folder.
If it fails
The usual reasons:
  • SERVER_DIR is wrong
  • WORLD_DIR is wrong
  • You accidentally saved it as .bat.txt

Step 7, Create cleanup script, deletes backups after 48 hours

Create a new file and save it as:
C:\MinecraftTools\cleanup_backups.bat
Paste this: 
@echo off
setlocal
set DEST=C:\MinecraftBackups

REM Delete backups older than 2 days (48 hours)
forfiles /p "%DEST%" /d -2 /c "cmd /c rd /s /q @path"

endlocal
Test it now
Double click cleanup_backups.bat. It should not delete fresh backups.

Step 9, Schedule hourly backups from 08:00 to 22:00

We will create a task that runs the backup script every hour, only during open hours.

Create the task

  1. Click Task Scheduler Library
  2. In the right Actions panel, click Create Task

General tab

  • Name: Minecraft Hourly Backup
  • Tick Run whether user is logged on or not
  • Tick Run with highest privileges

Triggers tab

  1. Click New
  2. Begin the task: On a schedule
  3. Daily
  4. Start: 08:00
  5. Tick Repeat task every: 1 hour
  6. For a duration of: 14 hours
  7. Click OK

Actions tab

  1. Click New
  2. Action: Start a program
  3. Click Browse
  4. Select C:\MinecraftTools\backup_world.bat
  5. Click OK

Finish and test

Click OK to create the task.

Test:
  1. Find Minecraft Hourly Backup in the middle list
  2. Right click it, click Run
  3. Check C:\MinecraftBackups for a new backup folder

Step 10, Schedule cleanup daily

This deletes backups older than 48 hours so the disk does not fill up.

Create the cleanup task

  1. Click Task Scheduler Library
  2. Click Create Task

General tab

  • Name: Minecraft Backup Cleanup
  • Tick Run whether user is logged on or not
  • Tick Run with highest privileges

Triggers tab

  1. Click New
  2. On a schedule
  3. Daily
  4. Start: 07:50
  5. OK

Actions tab

  1. Click New
  2. Start a program
  3. Browse to C:\MinecraftTools\cleanup_backups.bat
  4. OK
Test it
Right click the task, Run. It should run without errors.

Step 11, Whitelist and no operator powers

No operators

In the server folder, open ops.json and make sure it is: 
[]

Whitelist

In server.properties make sure: 
white-list=true
When the server is running, add friends in the server console: 
whitelist add PlayerName
Remove someone who breaks rules: 
whitelist remove PlayerName
Backups fix damage. Whitelist stops repeat offenders coming back.

Step 12, Restore after griefing

Restore steps:
  1. Stop the server
  2. Open C:\MinecraftBackups
  3. Pick a backup folder from before the damage
  4. Rename current world to world_broken
  5. Copy backup folder into the server folder
  6. Rename copied backup folder to world
  7. Start the server
Always stop the server before copying world files.
Copying while it runs can corrupt the world.
Tip: The only place you must edit paths is in Step 6, the backup script.
